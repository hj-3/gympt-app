   ```bash

   .env

   secrets.yaml
   *-secrets.yaml
   ```

2. ❌ **Kubernetes Secret manifest에 실제 값 하드코딩 금지**
   ```yaml
   # BAD - 절대 이렇게 하지 마세요!
   apiVersion: v1
   kind: Secret
   metadata:
     name: db-password
   data:
     password: bXlwYXNzd29yZA==  # base64 encoded, but still visible in Git!
   ```

3. ❌ **소스 코드에 Secret 하드코딩 금지**
   ```java
   // BAD
   String dbPassword = "mypassword123";
   
   // GOOD
   String dbPassword = System.getenv("DB_PASSWORD");
   ```

## 로컬 환경 (.env)

### .env 파일 사용법

로컬 개발 환경에서는 `.env` 파일을 사용합니다.

#### 1. .env 파일 생성

```bash
# 루트 디렉토리
cd gympt-app
cp .env.example .env

# 개별 서비스
cd backend-api
cp .env.example .env
```

#### 2. .env 파일 편집

```bash
# 텍스트 에디터로 .env 파일 열기
vim .env
# 또는
code .env
```

```env
# .env (로컬 개발용)
APP_ENV=local
DB_HOST=localhost
DB_PORT=5432
DB_NAME=gympt
DB_USERNAME=gympt_user
DB_PASSWORD=local_dev_password  # 로컬에서는 약한 비밀번호 사용 가능

# LocalStack 사용
S3_ENDPOINT_URL=http://localhost:4566
SQS_ENDPOINT_URL=http://localhost:4566
DYNAMODB_ENDPOINT_URL=http://localhost:4566
```

#### 3. .env 파일이 .gitignore에 포함되어 있는지 확인

```bash
# .gitignore 확인
cat .gitignore | grep ".env"

# 출력 예시:
# .env
# .env.local
# .env.*.local
# *.env
# !.env.example
```

#### 4. 환경변수 로드 확인

**Spring Boot (Java):**
```yaml
# application.yml
spring:
  datasource:
    password: ${DB_PASSWORD:default}  # 환경변수에서 로드
```

**FastAPI (Python):**
```python
# config.py
from pydantic_settings import BaseSettings

class Settings(BaseSettings):
    DB_PASSWORD: str
    
    class Config:
        env_file = ".env"

settings = Settings()  # 자동으로 .env 파일 로드
```

### .env.example 관리법

`.env.example`은 팀원들이 참고할 수 있도록 Git에 커밋됩니다.

#### 규칙

1. **실제 Secret 값을 넣지 않음**
   ```env
   # GOOD - .env.example
   DB_PASSWORD=changeme_local_password
   JWT_SECRET=change_this_secret_in_production
   SLACK_WEBHOOK_URL=                     # Empty, will be filled by user
   
   # BAD - .env.example
   DB_PASSWORD=actual_production_password  # 실제 값 노출!
   ```

2. **모든 필수 환경변수 포함**
   ```env
   # 모든 서비스에 필요한 환경변수를 나열
   APP_ENV=local
   DB_HOST=localhost
   DB_PASSWORD=changeme
   # ... 계속
   ```

3. **주석으로 설명 추가**
   ```env
   # Database Configuration
   DB_HOST=localhost                      # Use RDS endpoint in AWS
   DB_PASSWORD=changeme                   # Use strong password in production
   
   # Feature Flags
   FEATURE_AI_AGENT_ENABLED=true          # Enable AI Agent feature
   ```

4. **환경별 예시 값 제공**
   ```env
   # Local: http://localhost:4566
   # Dev: https://s3.ap-northeast-2.amazonaws.com
   # Prod: https://s3.ap-northeast-2.amazonaws.com
   S3_ENDPOINT_URL=http://localhost:4566
   ```

### .env 파일 업데이트 프로세스

새로운 환경변수가 추가될 때:

```bash
# 1. .env.example 업데이트
echo "NEW_API_KEY=" >> .env.example

# 2. Git 커밋
git add .env.example
git commit -m "docs: Add NEW_API_KEY to .env.example"

# 3. 팀원들에게 알림
# Slack 등을 통해 새 환경변수 추가를 알림

# 4. 각자 로컬 .env 파일 업데이트
echo "NEW_API_KEY=my_local_key" >> .env
```

## AWS 환경 (dev/prod)

AWS 환경에서는 **AWS Secrets Manager**와 **AWS Systems Manager Parameter Store**를 사용합니다.

### AWS Secrets Manager 사용 기준

**민감한 정보**는 Secrets Manager에 저장:

- ✅ Database 비밀번호
- ✅ JWT secret keys
- ✅ API keys (third-party services)
- ✅ Redis 비밀번호
- ✅ Slack webhook URLs
- ✅ 암호화 키

#### Secret 생성 예시

```bash
# Database password
aws secretsmanager create-secret \
  --name /gympt/dev/backend/db-password \
  --secret-string '{
    "password": "super_secure_password_123!"
  }' \
  --region ap-northeast-2

# JWT secret
aws secretsmanager create-secret \
  --name /gympt/dev/backend/jwt-secret \
  --secret-string '{
    "secret": "256_bit_random_string_for_jwt_signing"
  }' \
  --region ap-northeast-2

# Redis auth token
aws secretsmanager create-secret \
  --name /gympt/dev/redis/auth-token \
  --secret-string '{
    "password": "redis_secure_password"
  }' \
  --region ap-northeast-2

# Slack webhook
aws secretsmanager create-secret \
  --name /gympt/dev/slack/webhook-url \
  --secret-string '{
    "url": "https://hooks.slack.com/services/T00/B00/XXXX"
  }' \
  --region ap-northeast-2
```

#### Secret 조회

```bash
# Secret 값 확인
aws secretsmanager get-secret-value \
  --secret-id /gympt/dev/backend/db-password \
  --region ap-northeast-2 \
  --query SecretString \
  --output text | jq -r '.password'
```

#### Secret 업데이트

```bash
# Secret 값 업데이트
aws secretsmanager update-secret \
  --secret-id /gympt/dev/backend/db-password \
  --secret-string '{
    "password": "new_password_after_rotation"
  }' \
  --region ap-northeast-2
```

#### Secret 로테이션

```bash
# 자동 로테이션 설정
aws secretsmanager rotate-secret \
  --secret-id /gympt/dev/backend/db-password \
  --rotation-lambda-arn arn:aws:lambda:ap-northeast-2:ACCOUNT:function:rotate-db-password \
  --rotation-rules AutomaticallyAfterDays=30 \
  --region ap-northeast-2
```

### SSM Parameter Store 사용 기준

**비민감한 설정 값**은 Parameter Store에 저장:

- ✅ AWS 리소스 이름 (S3 bucket, SQS queue URL)
- ✅ AWS region
- ✅ Feature flags
- ✅ 공개 설정 값

#### Parameter 생성 예시

```bash
# S3 bucket name
aws ssm put-parameter \
  --name /gympt/dev/s3/media-bucket \
  --value gympt-media-dev \
  --type String \
  --region ap-northeast-2

# SQS queue URL
aws ssm put-parameter \
  --name /gympt/dev/sqs/posture-event-queue-url \
  --value https://sqs.ap-northeast-2.amazonaws.com/123456789012/gympt-posture-events-dev \
  --type String \
  --region ap-northeast-2

# EventBridge bus name
aws ssm put-parameter \
  --name /gympt/dev/eventbridge/bus-name \
  --value gympt-events-dev \
  --type String \
  --region ap-northeast-2

# Bedrock configuration
aws ssm put-parameter \
  --name /gympt/dev/bedrock/model-id \
  --value anthropic.claude-3-5-sonnet-20241022-v2:0 \
  --type String \
  --region ap-northeast-2
```

#### Parameter 조회

```bash
# Parameter 값 확인
aws ssm get-parameter \
  --name /gympt/dev/s3/media-bucket \
  --region ap-northeast-2 \
  --query Parameter.Value \
  --output text
```

### Secrets Manager vs Parameter Store 비교

| 항목 | Secrets Manager | Parameter Store |
|------|----------------|----------------|
| **용도** | 민감한 정보 | 비민감한 설정 |
| **자동 로테이션** | ✅ 지원 | ❌ 미지원 |
| **버전 관리** | ✅ 자동 | ✅ 수동 |
| **암호화** | ✅ 기본 암호화 | ⚠️ SecureString 사용 필요 |
| **비용** | $0.40/secret/month | 무료 (Standard) |
| **예시** | DB password, API keys | S3 bucket names, URLs |

## External Secrets Operator

External Secrets Operator는 AWS Secrets Manager/Parameter Store의 Secret을 Kubernetes Secret으로 자동 동기화합니다.

### 동작 방식

```
┌─────────────────────────────────────────────────────────────┐
│               AWS Secrets Manager                            │
│                                                              │
│  /gympt/dev/backend/db-password: "mysecret123"             │
│  /gympt/dev/backend/jwt-secret: "jwt_secret_key"           │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           │ IRSA (IAM Role for Service Account)
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│          External Secrets Operator (Kubernetes)             │
│                                                              │
│  ExternalSecret Resource:                                   │
│  - secretStoreRef: aws-secrets-manager                      │
│  - data:                                                     │
│      - secretKey: DB_PASSWORD                               │
│        remoteRef:                                            │
│          key: /gympt/dev/backend/db-password                │
│          property: password                                  │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           │ Creates/Syncs
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│              Kubernetes Secret (Created)                     │
│                                                              │
│  apiVersion: v1                                             │
│  kind: Secret                                               │
│  metadata:                                                  │
│    name: backend-api-secrets                               │
│  data:                                                      │
│    DB_PASSWORD: bXlzZWNyZXQxMjM=  # base64 encoded         │
│    JWT_SECRET: and0X3NlY3JldF9rZXk=                       │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           │ Mounted as env vars
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│                Application Pod                               │
│                                                              │
│  env:                                                        │
│    - name: DB_PASSWORD                                      │
│      valueFrom:                                             │
│        secretKeyRef:                                        │
│          name: backend-api-secrets                         │
│          key: DB_PASSWORD                                   │
└─────────────────────────────────────────────────────────────┘
```

### ExternalSecret 리소스 생성

ExternalSecret은 **GitOps 레포**에서 관리하며, **실제 Secret 값은 포함하지 않습니다**.

```yaml
# gympt-gitops/apps/backend-api/externalsecret-dev.yaml
apiVersion: external-secrets.io/v1beta1
kind: ExternalSecret
metadata:
  name: backend-api-secrets
  namespace: gympt-dev
spec:
  refreshInterval: 1h  # 1시간마다 AWS에서 동기화

  # ClusterSecretStore 참조
  secretStoreRef:
    name: aws-secrets-manager
    kind: ClusterSecretStore

  # 생성할 Kubernetes Secret 이름
  target:
    name: backend-api-secrets
    creationPolicy: Owner

  # AWS Secrets Manager → Kubernetes Secret 매핑
  data:
    # DB_PASSWORD라는 key로 Kubernetes Secret 생성
    - secretKey: DB_PASSWORD
      remoteRef:
        # AWS Secrets Manager의 secret 이름
        key: /gympt/dev/backend/db-password
        # JSON 내의 property 이름
        property: password

    - secretKey: JWT_SECRET
      remoteRef:
        key: /gympt/dev/backend/jwt-secret
        property: secret
```

### ClusterSecretStore 설정

ClusterSecretStore는 AWS Secrets Manager에 접근하는 방법을 정의합니다.

```yaml
# gympt-gitops/platform/external-secrets/cluster-secret-store.yaml
apiVersion: external-secrets.io/v1beta1
kind: ClusterSecretStore
metadata:
  name: aws-secrets-manager
spec:
  provider:
    aws:
      service: SecretsManager
      region: ap-northeast-2
      auth:
        jwt:  # IRSA 사용
          serviceAccountRef:
            name: external-secrets-sa
            namespace: external-secrets-system
```

### ServiceAccount with IRSA

IRSA (IAM Roles for Service Accounts)를 사용하여 AWS 권한 부여:

```yaml
# ServiceAccount
apiVersion: v1
kind: ServiceAccount
metadata:
  name: external-secrets-sa
  namespace: external-secrets-system
  annotations:
    # IAM Role ARN (Terraform에서 생성)
    eks.amazonaws.com/role-arn: arn:aws:iam::123456789012:role/gympt-external-secrets-role
```

Terraform에서 IAM Role 생성:

```hcl
# Terraform
resource "aws_iam_role" "external_secrets" {
  name = "gympt-external-secrets-role"

  assume_role_policy = jsonencode({
    Version = "2012-10-17"
    Statement = [{
      Effect = "Allow"
      Principal = {
        Federated = "arn:aws:iam::${var.account_id}:oidc-provider/${var.oidc_provider}"
      }
      Action = "sts:AssumeRoleWithWebIdentity"
      Condition = {
        StringEquals = {
          "${var.oidc_provider}:sub" = "system:serviceaccount:external-secrets-system:external-secrets-sa"
        }
      }
    }]
  })
}

resource "aws_iam_role_policy" "external_secrets" {
  role = aws_iam_role.external_secrets.id

  policy = jsonencode({
    Version = "2012-10-17"
    Statement = [{
      Effect = "Allow"
      Action = [
        "secretsmanager:GetSecretValue",
        "secretsmanager:DescribeSecret"
      ]
      Resource = "arn:aws:secretsmanager:ap-northeast-2:${var.account_id}:secret:/gympt/${var.environment}/*"
    }, {
      Effect = "Allow"
      Action = [
        "ssm:GetParameter",
        "ssm:GetParameters"
      ]
      Resource = "arn:aws:ssm:ap-northeast-2:${var.account_id}:parameter/gympt/${var.environment}/*"
    }]
  })
}
```

### ExternalSecret 사용 흐름

1. **Terraform으로 AWS Secret 생성**
   ```bash
   cd gympt-infra/terraform/environments/dev
   terraform apply
   ```

2. **AWS에 실제 Secret 값 저장**
   ```bash
   aws secretsmanager create-secret \
     --name /gympt/dev/backend/db-password \
     --secret-string '{"password":"actual_password"}'
   ```

3. **GitOps 레포에 ExternalSecret 정의**
   ```bash
   cd gympt-gitops
   # externalsecret-dev.yaml 생성 (실제 값 없음!)
   git add apps/backend-api/externalsecret-dev.yaml
   git commit -m "feat: Add ExternalSecret for backend-api"
   git push
   ```

4. **Argo CD가 자동으로 배포**
   ```bash
   # Argo CD가 ExternalSecret을 클러스터에 적용
   # External Secrets Operator가 자동으로 Kubernetes Secret 생성
   ```

5. **확인**
   ```bash
   # Kubernetes Secret이 자동 생성되었는지 확인
   kubectl get externalsecret -n gympt-dev
   kubectl get secret backend-api-secrets -n gympt-dev
   
   # Secret 값 확인 (base64 디코딩)
   kubectl get secret backend-api-secrets -n gympt-dev -o jsonpath='{.data.DB_PASSWORD}' | base64 -d
   ```

### Secret 업데이트 흐름

```bash
# 1. AWS Secrets Manager에서 Secret 업데이트
aws secretsmanager update-secret \
  --secret-id /gympt/dev/backend/db-password \
  --secret-string '{"password":"new_password"}'

# 2. External Secrets Operator가 자동으로 감지 (refreshInterval: 1h)
# 또는 강제 리프레시
kubectl annotate externalsecret backend-api-secrets \
  force-sync=$(date +%s) --overwrite -n gympt-dev

# 3. Pod 재시작 (필요 시)
kubectl rollout restart deployment/backend-api -n gympt-dev
```

## GitHub Actions OIDC

### GitHub Actions OIDC 사용 이유

**문제: AWS Access Keys는 보안 위험**

```yaml
# BAD - Access Keys를 GitHub Secrets에 저장
- name: Configure AWS Credentials
  uses: aws-actions/configure-aws-credentials@v4
  with:
    aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
    aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
```

**문제점:**
- ❌ Access Keys가 유출될 위험
- ❌ Key 로테이션이 어려움
- ❌ GitHub Secrets에 저장 필요
- ❌ 권한 관리가 복잡

**해결책: OIDC 사용**

```yaml
# GOOD - OIDC 사용 (No Access Keys!)
- name: Configure AWS Credentials
  uses: aws-actions/configure-aws-credentials@v4
  with:
    role-to-assume: arn:aws:iam::123456789012:role/github-actions-role
    aws-region: ap-northeast-2
```

**장점:**
- ✅ No Access Keys!
- ✅ 임시 자격증명 사용
- ✅ IAM Role로 권한 관리
- ✅ GitHub Secrets 최소화

### OIDC 설정 방법

#### 1. AWS에서 OIDC Provider 생성 (Terraform)

```hcl
# terraform/modules/github-oidc/main.tf
resource "aws_iam_openid_connect_provider" "github" {
  url = "https://token.actions.githubusercontent.com"

  client_id_list = ["sts.amazonaws.com"]

  thumbprint_list = [
    "6938fd4d98bab03faadb97b34396831e3780aea1"
  ]
}

resource "aws_iam_role" "github_actions" {
  name = "github-actions-role"

  assume_role_policy = jsonencode({
    Version = "2012-10-17"
    Statement = [{
      Effect = "Allow"
      Principal = {
        Federated = aws_iam_openid_connect_provider.github.arn
      }
      Action = "sts:AssumeRoleWithWebIdentity"
      Condition = {
        StringEquals = {
          "token.actions.githubusercontent.com:aud" = "sts.amazonaws.com"
        }
        StringLike = {
          "token.actions.githubusercontent.com:sub" = "repo:your-org/gympt-app:*"
        }
      }
    }]
  })
}

resource "aws_iam_role_policy_attachment" "github_actions_ecr" {
  role       = aws_iam_role.github_actions.name
  policy_arn = "arn:aws:iam::aws:policy/AmazonEC2ContainerRegistryPowerUser"
}
```

#### 2. GitHub Actions Workflow에서 사용

```yaml
# .github/workflows/deploy.yml
name: Deploy

on:
  push:
    branches: [main]

permissions:
  id-token: write  # OIDC token 발급에 필요
  contents: read

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: arn:aws:iam::123456789012:role/github-actions-role
          aws-region: ap-northeast-2
          # No aws-access-key-id!
          # No aws-secret-access-key!

      - name: Login to ECR
        id: login-ecr
        uses: aws-actions/amazon-ecr-login@v2

      - name: Build and push
        run: |
          docker build -t ${{ steps.login-ecr.outputs.registry }}/gympt/backend-api:${GITHUB_SHA::7} .
          docker push ${{ steps.login-ecr.outputs.registry }}/gympt/backend-api:${GITHUB_SHA::7}
```

```yaml
SLACK_WEBHOOK_URL: https://hooks.slack.com/services/xxx
```

**저장하지 않는 것:**
- ❌ AWS_ACCESS_KEY_ID
- ❌ AWS_SECRET_ACCESS_KEY
- ❌ Database passwords
- ❌ JWT secrets

## Secret을 GitOps 레포에 넣으면 안 되는 이유

### 문제 상황

```yaml
# BAD - gympt-gitops/apps/backend-api/secret.yaml
apiVersion: v1
kind: Secret
metadata:
  name: backend-api-secrets
data:
  DB_PASSWORD: bXlwYXNzd29yZDEyMw==  # base64("mypassword123")
  JWT_SECRET: and0X3NlY3JldF9rZXk=    # base64("jwt_secret_key")
```

### 왜 위험한가?

1. **Git 히스토리에 영구 저장**
   ```bash
   # Secret을 삭제해도 Git 히스토리에 남음
   git log --all --full-history -- "*secret*"
   ```

2. **Base64는 암호화가 아님**
   ```bash
   # 누구나 디코딩 가능
   echo "bXlwYXNzd29yZDEyMw==" | base64 -d
   # Output: mypassword123
   ```

3. **접근 제어 어려움**
   - Git 레포에 접근 가능한 모든 사람이 Secret 확인 가능
   - 감사 로그 부족

4. **로테이션 어려움**
   - Secret 변경 시 Git 커밋 필요
   - CI/CD 파이프라인 트리거

### 올바른 방법

```yaml
# GOOD - gympt-gitops/apps/backend-api/externalsecret-dev.yaml
# 실제 Secret 값은 AWS Secrets Manager에 저장
# GitOps 레포에는 참조만 저장
apiVersion: external-secrets.io/v1beta1
kind: ExternalSecret
metadata:
  name: backend-api-secrets
spec:
  data:
    - secretKey: DB_PASSWORD
      remoteRef:
        key: /gympt/dev/backend/db-password  # AWS Secrets Manager path
        property: password

# GitOps 레포에 실제 값 없음! ✅
```

## 트러블슈팅

### 1. .env 파일이 로드되지 않음

**증상:**
```
Error: DB_PASSWORD environment variable not found
```

**해결:**
```bash
# 1. .env 파일 존재 확인
ls -la .env

# 2. 환경변수 로드 확인
source .env
env | grep DB_PASSWORD

# 3. 애플리케이션 재시작
./gradlew bootRun  # Spring Boot
python -m uvicorn app.main:app  # FastAPI
```

### 2. External Secrets Operator가 동작하지 않음

**증상:**
```bash
kubectl get externalsecret backend-api-secrets -n gympt-dev
# Status: SecretSyncedError
```

**해결:**
```bash
# 1. ExternalSecret 상태 확인
kubectl describe externalsecret backend-api-secrets -n gympt-dev

# 2. External Secrets Operator 로그 확인
kubectl logs -n external-secrets-system deployment/external-secrets

# 3. IRSA 권한 확인
kubectl describe sa external-secrets-sa -n external-secrets-system
# eks.amazonaws.com/role-arn annotation 확인

# 4. AWS Secrets Manager에 Secret 존재 확인
aws secretsmanager get-secret-value \
  --secret-id /gympt/dev/backend/db-password \
  --region ap-northeast-2

# 5. 강제 리프레시
kubectl annotate externalsecret backend-api-secrets \
  force-sync=$(date +%s) --overwrite -n gympt-dev
```

### 3. GitHub Actions OIDC 인증 실패

**증상:**
```
Error: Unable to assume role arn:aws:iam::xxx:role/github-actions-role
```

**해결:**
```bash
# 1. IAM Role Trust Relationship 확인
aws iam get-role --role-name github-actions-role

# 2. OIDC Provider 확인
aws iam list-open-id-connect-providers

# 3. GitHub Actions workflow permissions 확인
# permissions:
#   id-token: write  # 필수!
#   contents: read
```

### 4. Secret이 Pod에 반영되지 않음

**증상:**
Pod에서 환경변수가 비어있음

**해결:**
```bash
# 1. Secret 생성 확인
kubectl get secret backend-api-secrets -n gympt-dev

# 2. Secret 값 확인
kubectl get secret backend-api-secrets -n gympt-dev -o yaml

# 3. Deployment에서 Secret 참조 확인
kubectl get deployment backend-api -n gympt-dev -o yaml | grep -A 10 envFrom

# 4. Pod 재시작
kubectl rollout restart deployment/backend-api -n gympt-dev

# 5. Pod 환경변수 확인
kubectl exec -it deployment/backend-api -n gympt-dev -- env | grep DB_PASSWORD
```

### 5. 로컬에서 AWS 서비스 접근 실패

**증상:**
```
botocore.exceptions.EndpointConnectionError: Could not connect to the endpoint URL
```

**해결:**
```bash
# LocalStack 사용 확인
echo $S3_ENDPOINT_URL
# Output: http://localhost:4566

# LocalStack 실행 확인
docker ps | grep localstack

# LocalStack 로그 확인
docker logs gympt-localstack

# AWS CLI로 LocalStack 테스트
aws --endpoint-url=http://localhost:4566 s3 ls
```
