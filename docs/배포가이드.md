## 배포 개요

### 배포 아키텍처

```
GitHub (gympt-app) → GitHub Actions → ECR → ArgoCD → EKS
                                              ↓
                            gympt-gitops (Kubernetes manifests)
```

### 3-레포지토리 구조

1. **gympt-app** (이 레포지토리)
   - 애플리케이션 소스 코드
   - Dockerfiles
   - GitHub Actions workflows

2. **gympt-infra**
   - Terraform 코드
   - AWS 인프라 정의

3. **gympt-gitops**
   - Kubernetes manifests
   - Helm charts
   - ArgoCD Applications

### 배포 전략

- **dev 환경:** 자동 배포 (develop 브랜치 푸시 시)
- **prod 환경:** 수동 승인 후 배포 (main 브랜치 PR 머지 후)
- **배포 방식:** Rolling Update (무중단 배포)
- **카나리 배포:** 중요 변경 시 선택적 적용

## CI/CD 파이프라인

### Backend API 배포 파이프라인

**Workflow:** `.github/workflows/backend-api-ci.yml`

**트리거:**
- `push` to `main` or `develop`
- `pull_request` to `main` (테스트만 실행)
- Paths: `backend-api/**`

**단계:**

1. **Build & Test**
   ```yaml
   - Checkout code
   - Setup JDK 21
   - Gradle build
   - Run tests (JUnit)
   - Generate coverage report (JaCoCo)
   ```

2. **Docker Build & Push**
   ```yaml
   - Login to Amazon ECR
   - Build Docker image
   - Tag: {env}-{git_sha} and {env}-latest
   - Push to ECR
   ```

3. **Update GitOps Repo**
   ```yaml
   - Clone gympt-gitops
   - Update image tag in values-{env}.yaml
   - Commit and push
   ```

4. **Notify**
   ```yaml
   - Send Slack notification
   - Update GitHub commit status
   ```

**실행 예시:**

```bash
# develop 브랜치에 푸시
git checkout develop
git add backend-api/
git commit -m "feat(backend): 사용자 인증 개선"
git push origin develop

# GitHub Actions 자동 실행
# 1. 테스트 (약 3분)
# 2. Docker 빌드 & ECR 푸시 (약 5분)
# 3. GitOps 업데이트 (약 30초)
# 4. ArgoCD 자동 동기화 (약 2분)
# 총 소요 시간: 약 10-12분
```

### Python Services (Agent, Posture Analysis) 파이프라인

**Workflow:** `.github/workflows/agent-service-ci.yml`

**단계:**

1. **Test**
   ```yaml
   - Setup Python 3.12
   - Install dependencies
   - Run pytest
   - Generate coverage report
   ```

2. **Docker Build & Push**
   ```yaml
   - Build multi-stage Dockerfile
   - Push to ECR
   ```

3. **Update GitOps**
   ```yaml
   - Update Helm values
   - Trigger ArgoCD sync
   ```

### Lambda Functions 배포

**Workflow:** `.github/workflows/lambda-deploy.yml`

**트리거:**
- Paths: `lambdas/**/*.py`

**단계:**

1. **Package**
   ```bash
   # 각 Lambda별로 패키징
   cd lambdas/report-generator
   pip install -r requirements.txt -t package/
   cd package && zip -r ../function.zip .
   cd .. && zip -g function.zip *.py
   ```

2. **Deploy to AWS Lambda**
   ```yaml
   - aws lambda update-function-code \
       --function-name gympt-{env}-report-generator \
       --zip-file fileb://function.zip
   ```

3. **Update Aliases**
   ```yaml
   # Canary deployment for prod
   - aws lambda update-alias \
       --function-name gympt-prod-report-generator \
       --name live \
       --routing-config AdditionalVersionWeights={"$LATEST"=0.1}
   ```

### Frontend 배포 (S3 + CloudFront)

**Workflow:** `.github/workflows/frontend-deploy.yml`

**단계:**

1. **Build**
   ```bash
   cd frontend
   npm install
   npm run build:prod
   npm run export  # Static export
   ```

2. **Deploy to S3**
   ```bash
   aws s3 sync out/ s3://gympt-{env}-frontend/ \
     --delete \
     --cache-control "public, max-age=31536000, immutable" \
     --exclude "*.html"
   
   aws s3 sync out/ s3://gympt-{env}-frontend/ \
     --cache-control "no-cache" \
     --include "*.html"
   ```

3. **Invalidate CloudFront**
   ```bash
   aws cloudfront create-invalidation \
     --distribution-id E1234567890ABC \
     --paths "/*"
   ```

## 환경별 배포

### Dev 환경 배포

**자동 배포 (develop 브랜치)**

```bash
# 1. 기능 개발 완료
git checkout develop
git merge feature/my-feature

# 2. 푸시 (자동 배포 트리거)
git push origin develop

# 3. GitHub Actions에서 자동 진행
# - 테스트 실행
# - Docker 이미지 빌드
# - ECR 푸시
# - GitOps 업데이트
# - ArgoCD 자동 sync

# 4. 배포 확인
kubectl get pods -n gympt-dev
kubectl logs -f deployment/backend-api -n gympt-dev
```

**배포 상태 확인:**

```bash
# ArgoCD UI
open https://argocd.gympt.com

# 또는 CLI
argocd app get gympt-dev-backend-api

# Kubernetes
kubectl get deployment backend-api -n gympt-dev
kubectl describe deployment backend-api -n gympt-dev
```

### Prod 환경 배포

**수동 승인 배포 (main 브랜치)**

**절차:**

1. **PR 생성 및 리뷰**
   ```bash
   git checkout -b release/v1.2.0
   git merge develop
   git push origin release/v1.2.0
   
   # GitHub에서 PR 생성: release/v1.2.0 → main
   # 리뷰어 지정 (최소 2명)
   ```

2. **PR 머지**
   ```bash
   # GitHub UI에서 "Squash and merge"
   # main 브랜치로 머지
   ```

3. **수동 배포 승인**
   ```bash
   # GitHub Actions에서 대기 중인 Deployment 승인
   # "Review deployments" 클릭
   # "prod" environment 체크
   # "Approve and deploy" 클릭
   ```

4. **배포 진행**
   ```yaml
   # GitHub Actions 자동 실행
   - Build & Test
   - Docker image push (prod-{sha}, prod-latest)
   - Update gympt-gitops (prod values)
   - Wait for ArgoCD sync
   ```

5. **배포 검증**
   ```bash
   # Health check
   curl https://api.gympt.com/actuator/health
   
   # Smoke test
   curl -X POST https://api.gympt.com/api/v1/auth/login \
     -H "Content-Type: application/json" \
     -d '{"email":"test@gympt.com","password":"test"}'
   
   # Metrics 확인
   # Grafana: https://grafana.gympt.com
   # - Request rate
   # - Error rate
   # - Latency (p50, p95, p99)
   ```

### Canary 배포 (선택적)

**중요 변경사항에 대해 Canary 배포 적용**

```yaml
# gympt-gitops/charts/backend-api/values-prod.yaml

replicaCount: 10

canary:
  enabled: true
  steps:
    - setWeight: 10  # 10% traffic
    - pause: {duration: 5m}
    - setWeight: 30
    - pause: {duration: 5m}
    - setWeight: 50
    - pause: {duration: 5m}
    - setWeight: 100
```

**수동 Canary 제어:**

```bash
# ArgoCD UI에서 Canary 진행 상황 모니터링
# 문제 발생 시 즉시 롤백

# 또는 CLI
argocd app rollback gympt-prod-backend-api
```

## 롤백 절차

### 긴급 롤백 (이전 버전으로 복구)

**1. ArgoCD를 통한 롤백**

```bash
# ArgoCD UI
# 1. Application 선택
# 2. "History and rollback" 클릭
# 3. 이전 버전 선택
# 4. "Rollback" 클릭

# 또는 CLI
argocd app rollback gympt-prod-backend-api <revision>

# 최근 버전 확인
argocd app history gympt-prod-backend-api
```

**2. Kubernetes Native 롤백**

```bash
# Deployment 롤백
kubectl rollout undo deployment/backend-api -n gympt-prod

# 특정 리비전으로 롤백
kubectl rollout undo deployment/backend-api --to-revision=3 -n gympt-prod

# 롤백 상태 확인
kubectl rollout status deployment/backend-api -n gympt-prod
```

**3. GitOps를 통한 롤백**

```bash
# gympt-gitops 레포에서 이전 커밋으로 revert
cd gympt-gitops
git log charts/backend-api/values-prod.yaml

git revert <commit-hash>
git push origin main

# ArgoCD 자동 sync로 이전 버전 배포
```

### Lambda 롤백

```bash
# 이전 버전으로 Alias 업데이트
aws lambda update-alias \
  --function-name gympt-prod-report-generator \
  --name live \
  --function-version 42  # 이전 안정 버전

# 또는 $PREVIOUS 사용
aws lambda get-alias \
  --function-name gympt-prod-report-generator \
  --name live
```

### Frontend 롤백

```bash
# S3에서 이전 버전 복원 (S3 Versioning 활용)
aws s3api list-object-versions \
  --bucket gympt-prod-frontend \
  --prefix index.html

# 특정 버전 복원
aws s3api copy-object \
  --copy-source gympt-prod-frontend/index.html?versionId=ABC123 \
  --bucket gympt-prod-frontend \
  --key index.html

# CloudFront 캐시 무효화
aws cloudfront create-invalidation \
  --distribution-id E1234567890ABC \
  --paths "/*"
```

## 모니터링

### 배포 후 체크리스트

- [ ] **Health Check 통과**
  - `/actuator/health` → UP
  - All pods Running

- [ ] **메트릭 정상**
  - Request rate 정상
  - Error rate < 1%
  - Latency p99 < 500ms

- [ ] **로그 확인**
  - No ERROR logs
  - No WARN logs (중요한 것)

- [ ] **Smoke Test 성공**
  - 주요 API 엔드포인트 테스트
  - 인증/인가 정상

### Grafana 대시보드

**주요 대시보드:**
- **GYMPT Overview**: 전체 시스템 상태
- **Backend API**: Spring Boot 메트릭
- **Agent Service**: FastAPI 메트릭
- **Database**: PostgreSQL, Redis 메트릭
- **Kubernetes**: 클러스터 리소스

**Alert 조건:**
- Error rate > 5% (5분 이상)
- Latency p99 > 1s (5분 이상)
- Pod restart count > 3 (10분 내)
- CPU usage > 80% (10분 이상)
- Memory usage > 85% (10분 이상)

### Slack 알림

**알림 채널:**
- `#gympt-deployments`: 배포 시작/완료
- `#gympt-alerts`: 장애 알림
- `#gympt-prod-alerts`: 프로덕션 긴급 알림

## 트러블슈팅

### 배포 실패 시

**1. GitHub Actions 실패**

```bash
# Workflow 로그 확인
# GitHub UI → Actions → 실패한 workflow 클릭

# 일반적인 원인:
# - 테스트 실패 → 로그 확인 후 수정
# - Docker 빌드 실패 → Dockerfile 문법 체크
# - ECR 푸시 실패 → AWS 권한 확인
```

**2. ArgoCD Sync 실패**

```bash
# ArgoCD UI에서 에러 메시지 확인
argocd app get gympt-prod-backend-api

# 일반적인 원인:
# - Kubernetes manifest 문법 오류
# - Image pull 실패 (ECR 접근 권한)
# - Resource limit 초과
```

**3. Pod CrashLoopBackOff**

```bash
# Pod 로그 확인
kubectl logs deployment/backend-api -n gympt-prod

# 일반적인 원인:
# - 환경 변수 누락 → Secret/ConfigMap 확인
# - Database 연결 실패 → Security Group 확인
# - OOMKilled → Resource limits 증가
```
