## 시스템 개요

GYMPT는 AI 기반 개인화 운동 트레이닝 플랫폼으로, 다음과 같은 핵심 기능을 제공합니다:

- **AI 추천**: AWS Bedrock Claude를 활용한 개인화 운동 추천
- **실시간 자세 분석**: MediaPipe 기반 자세 교정 피드백
- **운동 세션 관리**: 실시간 스트리밍 및 분석
- **리포트 생성**: 운동 성과 분석 및 시각화
- **웨어러블 연동**: 다양한 디바이스 데이터 통합

### 설계 원칙

1. **마이크로서비스 아키텍처 (MSA)**
   - 도메인별 서비스 분리
   - 독립적인 배포 및 확장
   - 장애 격리 (Fault Isolation)

2. **클라우드 네이티브**
   - Kubernetes 기반 컨테이너 오케스트레이션
   - AWS 관리형 서비스 활용
   - GitOps 기반 배포 자동화

3. **이벤트 드리븐**
   - 비동기 메시징 (SQS, EventBridge)
   - 느슨한 결합 (Loose Coupling)
   - 확장성 및 복원력

## 마이크로서비스 구성

### 1. Backend API (Spring Boot)

**역할:** 메인 REST API, 사용자/운동 데이터 관리

**기술 스택:**
- Language: Java 21
- Framework: Spring Boot 3.2, Spring Data JPA
- Database: PostgreSQL (RDS), Redis (ElastiCache)
- Security: Spring Security, JWT

**주요 기능:**
- 사용자 인증/인가 (JWT)
- 운동 계획 CRUD
- 인바디 데이터 관리
- 운동 세션 관리
- S3 연동 (이미지/비디오 업로드)

**엔드포인트:**
- `/api/v1/auth/*` - 인증
- `/api/v1/users/*` - 사용자
- `/api/v1/workouts/*` - 운동
- `/api/v1/sessions/*` - 세션
- `/api/v1/body-profiles/*` - 인바디

**배포:**
- EKS에 Deployment로 배포
- HPA: CPU 70% 기준 자동 스케일링
- Resource: 1 CPU, 2GB RAM

---

### 2. Agent Service (FastAPI)

**역할:** AI 기반 운동 추천 및 채팅

**기술 스택:**
- Language: Python 3.12
- Framework: FastAPI, Pydantic
- AI: AWS Bedrock (Claude 3 Sonnet)
- Database: DynamoDB (채팅 이력)

**주요 기능:**
- AI 운동 추천 생성
- 채팅 인터페이스
- 운동 계획 AI 분석
- 개인화 알고리즘

**엔드포인트:**
- `/api/v1/chat` - 채팅
- `/api/v1/recommendations` - 추천
- `/api/v1/analysis` - 분석

**배포:**
- EKS에 Deployment
- Resource: 0.5 CPU, 1GB RAM

---

### 3. Posture Analysis Service (FastAPI)

**역할:** 실시간 자세 분석 및 교정 피드백

**기술 스택:**
- Language: Python 3.12
- Framework: FastAPI
- ML: MediaPipe, OpenCV
- Streaming: Kinesis Video Streams, WebSocket
- GPU: NVIDIA T4 (선택적)

**주요 기능:**
- 실시간 포즈 감지 (MediaPipe)
- 운동별 자세 교정 로직
- KVS 스트림 처리
- WebSocket 실시간 피드백

**엔드포인트:**
- `/api/v1/analyze` - 자세 분석
- `/ws/session/{session_id}` - WebSocket 연결

**배포:**
- EKS에 Deployment (GPU Node)
- Resource: 2 CPU, 4GB RAM, 1 GPU (optional)

---

### 4. Remediation Worker (Python)

**역할:** 자동 장애 감지 및 복구

**기술 스택:**
- Language: Python 3.12
- Framework: FastAPI (Health API) + Background Worker
- Integrations: Kubernetes API, ArgoCD API, Slack API

**주요 기능:**
- EKS 클러스터 헬스 체크
- Pod 자동 재시작
- ArgoCD Sync 자동화
- Slack 알림

**배포:**
- EKS에 Deployment
- 주기: 5분마다 체크

---

### 5. Lambda Functions (6개)

#### report-generator
- **트리거:** SQS (세션 종료 이벤트)
- **기능:** 운동 리포트 PDF 생성
- **출력:** S3 업로드 + DynamoDB 메타데이터

#### posture-event-processor
- **트리거:** SQS (자세 분석 이벤트)
- **기능:** 자세 데이터 집계 및 저장

#### notification
- **트리거:** EventBridge (스케줄)
- **기능:** 푸시 알림, 이메일 발송

#### wearable-sync
- **트리거:** EventBridge (10분마다)
- **기능:** 웨어러블 디바이스 데이터 동기화

#### export
- **트리거:** API Gateway (On-demand)
- **기능:** 데이터 CSV/JSON 내보내기

#### agent-action
- **트리거:** SQS (AI 에이전트 액션)
- **기능:** 비동기 AI 작업 처리

## 데이터 플로우

### 1. 사용자 인증 플로우

```
Client → Backend API: POST /api/v1/auth/login
                         ↓
              JWT 생성 (Redis에 RefreshToken 저장)
                         ↓
Client ← JWT Access Token + Refresh Token
```

### 2. 운동 추천 플로우

```
Client → Backend API: GET /api/v1/workouts/recommend
                         ↓
         Backend → Agent Service: POST /api/v1/recommendations
                                      ↓
                          Agent Service → Bedrock (Claude)
                                      ↓
                          Bedrock → 개인화 추천 생성
                                      ↓
         Backend ← Agent Service ← 추천 결과
                         ↓
Client ← 추천 운동 계획
```

### 3. 실시간 자세 분석 플로우

```
Client (Camera) → WebRTC → KVS
                              ↓
                 Posture Analysis Service
                         ↓ (Subscribe KVS)
                   MediaPipe 처리
                         ↓
                   자세 분석 + 피드백
                         ↓
Client ← WebSocket ← 실시간 피드백
                         ↓
                    (비동기) SQS
                         ↓
              Lambda (posture-event-processor)
                         ↓
                    DynamoDB 저장
```

### 4. 리포트 생성 플로우

```
Backend API (세션 종료)
       ↓
    SQS 이벤트 발행
       ↓
Lambda (report-generator)
       ↓
DynamoDB + S3 조회 (세션 데이터)
       ↓
PDF 생성 (matplotlib, reportlab)
       ↓
S3 업로드 + DynamoDB 메타데이터 저장
       ↓
(선택적) Notification Lambda → 사용자 알림
```

## 인프라 구성

### EKS 클러스터

**구성:**
- Region: ap-northeast-2 (Seoul)
- Kubernetes: 1.28
- Node Groups:
  - **app-nodes**: 일반 워크로드 (t3.medium, 3-6 nodes)
  - **gpu-nodes**: Posture Analysis (g4dn.xlarge, 1-2 nodes)

**네임스페이스:**
- `gympt-dev`: 개발 환경
- `gympt-prod`: 프로덕션 환경
- `monitoring`: Prometheus, Grafana
- `argocd`: GitOps

### 네트워크

```
VPC: 10.0.0.0/16
├── Public Subnets (10.0.1.0/24, 10.0.2.0/24)
│   └── ALB, NAT Gateway
└── Private Subnets (10.0.11.0/24, 10.0.12.0/24)
    ├── EKS Nodes
    ├── RDS (10.0.21.0/24)
    └── ElastiCache (10.0.22.0/24)
```

### 보안

- **IRSA (IAM Roles for Service Accounts)**: 각 서비스별 최소 권한
- **External Secrets Operator**: AWS Secrets Manager 통합
- **Network Policies**: Pod 간 통신 제어
- **Security Groups**: 계층별 접근 제어

### 배포 전략

- **GitOps**: ArgoCD 기반 자동 배포
- **Rolling Update**: 무중단 배포
- **Canary Release**: 중요 변경 시 카나리 배포
